# 要件定義書 — Eigononikki Web/PWA アプリ

> このドキュメントは、既存の `eigononikki` プロジェクト（Cursor + markdown による英語日記学習ワークスペース）を、Web/PWA アプリへ拡張するための **要件定義書** です。
> 設計の詳細は [`design.md`](design.md) を参照してください。

---

## 1. 背景と目的

### 1.1 背景

- 学習者は UX デザイナー。IELTS 6.5 取得と、大学院での英語論文・ディスカッション・エッセイ執筆を目標としている。
- 現状は Cursor 上で AI と一緒に、**写経 → 穴埋め → 自由作文＋添削** の 3 フェーズで日記学習を進めている（詳細は [`../README.md`](../README.md)）。
- 日記本文・日本語訳・写経のポイント・Vocabulary は `diaries/YYYY-MM-DD.md` という markdown ファイルに蓄積されている。
- 一方で、**生成された日記を「使い倒す」フェーズが手薄** だった。具体的には:
  - 発音・リスニングの強化（読み上げ）
  - 蓄積された語彙の横断的な復習
  - 過去日記の翻訳力チェックによる長期記憶化

### 1.2 目的

このアプリは、上記の「**生成後の活用**」レイヤーを担う。

- **生成**: 引き続き Cursor で行う（このアプリのスコープ外）
- **写経**: 引き続き紙のノートで行う（このアプリのスコープ外）
- **活用**（読み上げ・単語帳・復習）: このアプリで完結させる

### 1.3 既存プロジェクトとの関係

```
eigononikki/                        ← この学習ワークスペース（ソース）
├── diaries/*.md                    ← Cursor で生成、git push
├── prompts/                        ← Cursor 用のプロンプト
└── app-spec/                       ← 本ドキュメント群

eigononikki-app/                    ← 別リポジトリ（実装）
└── (Next.js プロジェクト)          ← diaries/*.md を GitHub 経由で取得
```

## 2. ペルソナ・利用シーン

### 2.1 ペルソナ

| 項目 | 内容 |
| --- | --- |
| 名前 | メイカ |
| 職業 | UX デザイナー |
| 母語 | 日本語 |
| 英語レベル | 大学受験レベルで一度仕上げたあと長期ブランク、リスタート中。文法勘は徐々に戻ってくる、語彙の再活性化が課題。 |
| 学習目標 | IELTS 6.5（大学院留学） |
| 興味 | デザイン（UX・サービス・倫理）、ケア倫理、組織論、読書（小説・エッセイ・哲学）、短歌、日常観察 |
| 好むトーン | 静か・内省的・少し文学的。「すごい！」より「ふむ」。 |

### 2.2 主な利用シーン

| シーン | 場所 | 時間 | 主な操作 |
| --- | --- | --- | --- |
| 朝の写経タイム | 机の Mac の前 | 朝 15-30 分 | 日記詳細を開いて読み上げを聞きながら、紙のノートに写経する |
| すきま時間の単語復習 | スマホ（通勤・休憩・布団の中） | 5-10 分 | 単語帳のフラッシュカードを数枚めくる |

### 2.3 利用しない / 想定しないシーン

- 旅行先など、ネットがほぼ繋がらない環境での「初回利用」（オフラインキャッシュは「一度開いたものに限る」）
- 複数人での共同学習・添削（自分ひとり用）

## 3. スコープ

### 3.1 スコープ内（このアプリで作る）

- 日記の取得・表示（GitHub から markdown を fetch）
- 読み上げ（Web Speech API）
- 単語・熟語の集約と検索
- 単語チェッククイズ
- 過去日記の翻訳チェッククイズ
- 学習状態のローカル保存とエクスポート/インポート

### 3.2 スコープ外（このアプリでは作らない）

| 項目 | 理由 | 代替手段 |
| --- | --- | --- |
| AI による日記生成 | 既存の Cursor + プロンプト運用が成熟しており、コスト・複雑さの追加に見合わない | Cursor で生成 → git push |
| 写経用のエディタ機能 | 紙のノートに手書きすることに意味があるため | 手書き継続 |
| 自由作文の添削 | Cursor の方が文脈付きで添削しやすい | Cursor の Phase 3 プロンプト |
| 認証・複数ユーザー対応 | 自分ひとりで使うため | — |
| ソーシャル機能（シェア・コメント） | 学習の集中を妨げるため | — |

### 3.3 将来拡張の余地（今は作らないが、後で足せるよう設計する）

- AI API（OpenAI / Anthropic / Gemini）直接接続による「アプリ内で日記生成」
- クラウド DB（Supabase 等）による複数端末間の学習状態同期
- スマホネイティブ移植（Expo / React Native）
- PWA プッシュ通知による復習リマインダ

## 4. 機能要件

### FR-1: 日記閲覧

| ID | 要件 | 優先度 |
| --- | --- | --- |
| FR-1.1 | GitHub の `diaries/` ディレクトリから markdown 一覧を取得し、日付の新しい順で表示する | Must |
| FR-1.2 | 日記詳細画面で、英文・日本語訳・写経のポイント・Vocabulary を区切って表示する | Must |
| FR-1.3 | 日記の Phase（1 写経 / 2 穴埋め / 3 自由作文）をバッジとして表示する | Should |
| FR-1.4 | タイトル・Topic・本文に対する全文検索（クライアントサイド） | Should |
| FR-1.5 | 同日複数日記（`YYYY-MM-DD-2.md` のような派生）に対応 | Must |
| FR-1.6 | 一度開いた日記はオフラインでも再表示できる | Should |

### FR-2: 読み上げ（TTS）

| ID | 要件 | 優先度 |
| --- | --- | --- |
| FR-2.1 | 日記詳細画面に「読み上げ開始」ボタンを置き、英文全体を読み上げる | Must |
| FR-2.2 | 読み上げ中の文（または段落）をハイライト表示する | Must |
| FR-2.3 | 速度を 0.5x / 0.75x / 1.0x / 1.25x / 1.5x から選べる | Must |
| FR-2.4 | 音声（男声 / 女声、英米差）を端末で利用可能なものから選べる | Should |
| FR-2.5 | 一時停止・再開・スキップ（次の文へ）・ループ（同じ文を繰り返す）ができる | Should |
| FR-2.6 | 単語をタップすると、その単語だけを発音する | Could |

### FR-3: 単語帳

| ID | 要件 | 優先度 |
| --- | --- | --- |
| FR-3.1 | 全日記の `## Vocabulary` テーブルから単語・熟語を集約した一覧を表示する | Must |
| FR-3.2 | 同じ単語が複数日記に出てきた場合は 1 件にまとめ、出現回数と出典日を表示する | Must |
| FR-3.3 | 品詞・初出日・習得度でフィルタ／ソートできる | Should |
| FR-3.4 | 単語をタップすると、出典の例文と出典日記へのリンクが表示される | Must |
| FR-3.5 | 単語の発音（個別 TTS）ができる | Should |

### FR-4: 単語チェッククイズ

| ID | 要件 | 優先度 |
| --- | --- | --- |
| FR-4.1 | 単語帳から出題し、「覚えているか」をチェックできる | Must |
| FR-4.2 | 出題方向を選べる（英 → 日 / 日 → 英 / ランダム） | Must |
| FR-4.3 | 出題対象を選べる（全部 / 未習得のみ / 復習推奨日が来たもののみ） | Must |
| FR-4.4 | 結果に応じて習得度（SRS レベル）を更新する | Must |
| FR-4.5 | 1 セッションの問題数を設定できる（例: 10 問 / 20 問 / 無制限） | Should |
| FR-4.6 | セッション終了後にサマリ（正答率・新規習得語）を表示する | Should |
| FR-4.7 | **出題方式**: MVP は「表 → 裏フリップのみ」。設定画面から「4 択」モードに切り替えられる枠を将来用に残す（4 択ロジックは MVP では実装しない） | Must |

### FR-5: 過去日記の翻訳チェック

| ID | 要件 | 優先度 |
| --- | --- | --- |
| FR-5.1 | 復習推奨の過去日記から段落単位で出題する（日本語訳 → 英文を思い出す、または英文 → 日本語訳） | Must |
| FR-5.2 | ユーザーは「思い出せた / 微妙 / 思い出せなかった」を自己評価する | Must |
| FR-5.3 | 自己評価に応じて、その日記の次回復習日を更新する | Must |
| FR-5.4 | 段落ごとの正解（元の英文 / 日本語訳）を表示できる | Must |
| FR-5.5 | 「今日復習すべき日記」をホーム画面に表示する | Should |
| FR-5.6 | 復習リマインダはアプリ内バッジのみ（ホーム画面で「単語 N 個 / 日記 M 件」を表示）。PWA プッシュ通知は MVP では実装しない | Must |

### FR-6: 設定・データ管理

| ID | 要件 | 優先度 |
| --- | --- | --- |
| FR-6.1 | 接続先 GitHub リポジトリ（owner/repo/branch/path）を設定できる。デフォルトは `<owner>/eigononikki/main/diaries` | Must |
| FR-6.2 | 学習状態（IndexedDB の中身）を JSON にエクスポート / インポートできる | Must |
| FR-6.3 | TTS のデフォルト速度・音声を設定できる | Should |
| FR-6.4 | カラーテーマは **紙ノート風ライト基調 1 本**（オフホワイト〜クリーム色、暗色テキスト）。OS ダーク追従・テーマ切替は MVP では行わない | Must |
| FR-6.5 | キャッシュをクリアできる（GitHub から再取得） | Should |

## 5. 非機能要件

### 5.1 パフォーマンス

- 日記一覧の初回表示: 3 秒以内（日記 100 件程度を想定）
- 日記詳細画面の表示: 1 秒以内
- 単語クイズの遷移: 200ms 以内（体感のスムーズさを重視）

### 5.2 可用性・オフライン

- PWA としてホーム画面に追加できる
- 一度取得した日記はオフラインでも閲覧可能（Service Worker キャッシュ）
- ネットワーク断時に「オフラインで表示中」のインジケータを出す

### 5.3 データ所有権・プライバシー

- 学習状態（覚えた単語、復習履歴等）は **すべて端末ローカル**（IndexedDB）に保存する
- 外部サーバーに学習データを送信しない
- ユーザーはいつでも全データを JSON でエクスポートでき、別端末にインポートできる

### 5.4 アクセシビリティ

- キーボードのみで主要操作（クイズ回答、読み上げ操作）ができる
- フォントサイズを 3 段階で切り替えられる
- 読み上げ中の文ハイライトはコントラスト比 WCAG AA 以上

### 5.5 対応ブラウザ・端末

- デスクトップ: 最新版の Chrome / Safari / Firefox / Edge
- モバイル: iOS 16+ Safari、Android 最新 Chrome
- Web Speech API が動作しない環境では、読み上げ機能のみ無効化し、その他は動作させる

### 5.6 多言語対応

- UI 言語は日本語のみ（メイカさん本人用）
- コンテンツ（日記本文）は英語と日本語

---

## 6. 制約条件

- 月額コストはゼロを目標（Vercel 無料枠 + GitHub public repo + ローカルストレージ）
- 日記の markdown フォーマットは既存のものを変更しない（学習中の運用を壊さないため）
- GitHub repo は public とし、PAT 等の認証は不要にする（FR-6.1 のデフォルト動作）
- 開発期間は MVP まで **2-4 週末程度** を想定

## 7. 用語集

| 用語 | 意味 |
